# Dynatrace License Optimisation Workflow & Dashboard

## Overview

This repository contains a Dynatrace Workflow template and supporting Dashboard designed to help identify potential Dynatrace license optimisation opportunities across multiple licensing modules.

The solution queries recent license consumption patterns, analyses potential optimisation candidates, and generates an AI-assisted email report that summarises high-value opportunities for review. A supporting dashboard is also provided so users can drill into the underlying data and validate recommendations before making any changes.

The assessment is based on the last 30 days of available data.

## Purpose

The purpose of this solution is to help Dynatrace users and customer teams quickly identify areas where license consumption may be optimised without reducing observability value.

The Workflow and Dashboard are intended to support conversations around:

* License efficiency
* Consumption visibility
* Cost optimisation
* Monitoring configuration hygiene
* Customer value realisation
* FinOps and observability governance

This solution does **not** automatically change any Dynatrace configuration. It identifies optimisation candidates that should be reviewed and validated by the relevant service, platform, application, or observability owners.

## What It Analyses

The Workflow checks for several common license optimisation opportunities.

### Full-Stack Hosts with No Detected Services

Identifies hosts that are monitored in Full-Stack mode but have no detected monitored services during the assessment period.

Potential optimisation:

* Review whether Full-Stack monitoring is required.
* Consider switching suitable hosts to Infrastructure-only monitoring where application or service visibility is not required.

### Billable DQL Queries in Anomaly Detection

Identifies DQL queries against billable data types, such as logs, traces, and events, that are being executed by Anomaly Detection.

The data includes:

* Total scanned billable volume
* Associated anomaly detector
* Owner of the anomaly detector, where available

Potential optimisation:

* Add an execution delay where appropriate.
* Use OpenPipeline to convert frequently queried data into metrics where suitable, as metric queries do not incur query cost.

### Billable DQL Queries in Dashboards and Notebooks

Identifies DQL queries against billable logs, traces, and events that are being executed from Dashboards and Notebooks.

The Workflow checks these data types separately:

* Logs
* Traces
* Events

The data includes:

* Total scanned billable volume
* Dashboard or Notebook usage
* Number of users
* Number of views
* Dashboard links, where available

Potential optimisation:

* Review high-volume queries from frequently viewed dashboards.
* Convert frequently queried data into metrics using OpenPipeline where appropriate.
* Narrow query timeframes or optimise DQL queries where possible.

### Browser Synthetic Monitor Optimisation

Identifies Browser Synthetic monitors with potentially inefficient consumption patterns.

The data includes:

* Execution frequency
* Number of locations
* Number of steps per execution
* Total billable steps
* Availability rate

Potential optimisation:

* Reduce execution frequency for stable monitors with few or no failures.
* Reduce the number of execution locations where broad coverage is not required.
* Review monitors with extremely high failure rates, especially 90% or greater, as these may be misconfigured, obsolete, or testing unavailable endpoints.

### Potential Duplicate Synthetic Monitors

Identifies Synthetic monitors that appear to be testing the same URL.

Potential optimisation:

* Review monitors testing the same URL.
* Consolidate overlapping monitors where appropriate.
* Disable duplicate monitors where they do not provide unique monitoring value.

## Included Files

This repository includes:

* A Dynatrace Workflow template
* A Dynatrace Dashboard JSON file

The Dashboard provides supporting drill-down visibility for the findings generated by the Workflow.

The Workflow performs the data collection, analysis, and email report generation.

## Installation

### Prerequisites

Before installing, ensure you have:

* Access to a Dynatrace environment
* Permission to upload Dashboards
* Permission to upload and configure Workflows
* Access to the relevant licensing and consumption data in Dynatrace
* The required email recipients for the report

## Dashboard Installation

1. Download the Dashboard JSON file from this repository.

2. In Dynatrace, navigate to the **Dashboards** app.

3. Select **Upload**.

4. Upload the downloaded Dashboard JSON file.

5. Once uploaded, the dashboard will be available in your Dynatrace environment.

6. Update the dashboard rate card variables.

   These are free-text variables and should be updated to match the customer’s actual rate card values.

   The values should be copied 1:1 from the dollar cost values shown in the Dynatrace Account Management portal.

## Workflow Installation

1. Download the Workflow template from this repository.

2. In Dynatrace, navigate to the **Workflows** app.

3. Select **Upload**.

4. Upload the downloaded Workflow template.

5. Open the uploaded Workflow.

6. Navigate to the final Workflow step:

   `send_license_optimisation_report`

7. Update the email fields to include the relevant report recipients.

8. Save the Workflow.

9. Run the Workflow manually or configure an appropriate schedule. i.e. Change the trigger to a 30 day, repeating trigger

## Recommended Usage

This solution is intended to be used as a recurring review mechanism for license optimisation.

Recommended usage patterns include:

* Monthly license optimisation reviews
* Quarterly business reviews
* Customer value reviews
* FinOps reviews
* Internal Dynatrace health checks
* Pre-renewal or pre-expansion consumption analysis

The generated email report should be used as a high-level summary of potential optimisation opportunities. The supporting dashboard should be used to review and validate the underlying data before any action is taken.

## Important Notes

The findings generated by this solution are optimisation candidates only.

Before making changes, always validate:

* Whether the monitored entity is business-critical
* Whether monitoring mode changes could reduce required visibility
* Whether dashboard or notebook queries are still required
* Whether Synthetic monitors have a specific business, geographic, or operational purpose
* Whether duplicate-looking Synthetic monitors are actually testing different journeys, credentials, locations, or alerting requirements

Do not disable, modify, or downgrade monitoring without approval from the relevant owner.

## Output

The Workflow produces an AI-assisted email report that summarises the highest-value optimisation opportunities across the assessed licensing modules.

The report is designed to highlight:

* Key opportunity areas
* High-level consumption patterns
* Prioritised findings
* Suggested next actions
* Links to supporting dashboard detail

The email report intentionally avoids listing every raw finding. Instead, it summarises the most meaningful opportunities and directs users to the dashboard for deeper analysis.

## Disclaimer

This solution is intended to support Dynatrace license optimisation discussions and customer value conversations. It does not guarantee savings and does not automatically apply any configuration changes.

All recommendations should be reviewed and validated before implementation.
